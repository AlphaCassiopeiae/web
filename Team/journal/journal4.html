Content
Today, I tried to get software to successfully flash onto the Raspberry Pi pico. I followed the instructions found [here](https://datasheets.raspberrypi.com/pico/getting-started-with-pico.pdf) to install the raspberry pi pico extension in vs code.
Then, I ran the example code after storing it in a proper directory. The software rebooted several times, as seen below. One thing we figured out with the dev board software was that every time we wanted to load something, we needed to disconnect and reconnect the device while holding the BOOTSEL button on the Pi. We also figured out to not overload the IDE with several 'run' requests, or the device would get overloaded and not work. I also realized I needed a JDK in order to successfully compile the code. Although the main.c was running without this, I figured it would be essential to have in the future. I downloaded the JDK from [here](https://www.oracle.com/java/technologies/downloads/#java17).
I created an account an downloaded the JDK, and I proceeded to make a circuit to demonstrate that we have the ability to run any software we want. To test the circuit, I refreshed my knowledge of TFT tactile switches, by watching this [link](https://www.youtube.com/watch?v=0ZXYRU9KPG8) I constructed a circuit to demonstrate that the pico could flash, and then showed my work to Professor Walter. Professor Walter said we needed additional description on the PSDRs, so I plan to fine tune and revise those tonight
