Project Functional Description:

We plan to create a miniaturized version of the Altair 8800, one of the first microcomputers. The user interface for this device is a front panel with control switches and LEDs. This device in its complete form will allow the user to:

  1. Write programs to run on the machine using the front panel. The front panel will provide switches and LEDs such that the user can interact with memory, this way users can read and write Intel 8080 instructions to/from memory to be executed by the CPU card.<\li>
  2. Add optional expansion cards to allow more advanced functionality, for example, communication over UART. These cards will be extremely similar to the CPU and memory cards, so they will simply plug into the backplane PCB using DDR4 DIMM connectors. The user should be able to add/remove these optional cards to achieve the functionality they desire. For our project, we plan on providing a working UART card with custom firmware.

Components

Front Panel

The front panel will make up the user interface of our design. It will use LEDs to display information to the user, and that will consist of 8 status lights, 16 address bit lights, and 8 data bit lights. In terms of user control, the front panel will provide various slide switches for CPU control, memory examination, and reset functions.

Internal Cards

The enclosure houses a backplane PCB with 5 DDR4 DIMM connectors for functionality cards. These cards include CPU, memory, and UART cards, all based on the RP2350 microcontroller. All of these cards will have roughly the same layout, with the firmware/software driving the microcontrollers differentiating the functionality. These cards must be able to operate on the bus, which is heavily inspired by the original S100 bus used in the Altair. With the exception of the CPU and memory cards, the user should be able to add/remove other cards to have the functionality they want (just a UART card for our project).

Backplane PCB

Operation

Users will use the front panel to write, run, and debug i8080 programs. Programs can be entered manually using switches, stored in memory, and executed via the CPU.

Project Specific Design Requirements (PSDRs):

  1. (Software) Rust-based Intel 8080 processor emulation tied to an external clock.
  2. (Hardware) S-100-inspired 8080 bus protocol via RP2350 PIO with DDR4 DIMM connector.
  3. (Hardware) Switch and LED matrix for the front panel interfacing with the CPU.
  4. (Hardware) 5V to 3.3V step-down power supply using a reference design.
  5. (Hardware) Serial communication via a UART card with custom firmware.
