Project Functional Description:

We plan to create a miniaturized version of the Altair 8800, one of the first microcomputers. This device in its minimally complete form will allow the user to:
  1. Utilize the front panel to write, read, and execute Intel 8080 instructions in memory.
  2. Add expansion cards to allow the processor to read and write data over UART, to a disk, etc.


Components

The computer will be housed in an extruded aluminum enclosure. The front and back of the enclosure will be vertical PCBs that will house the main control interface on the front PCB and the power supply on the back PCB.

In the center of the front panel there are 16 address bit lights to indicate the address of the memory currently selected and above those and right-aligned, there are 8 data bit lights to show the value at that point in memory.

Below each of the 16 address bit lights there are 16 switches for manipulating the bits on the data and address lines.

And finally, there are 7 more switches below for turning on the computer, stopping/running the CPU, single stepping instructions, examining a value in a memory address specified on the address line, depositing a data value at an address, resetting the CPU, and protecting memory regions.

These will be connected to I/O expanders that will interface directly with one of the GPIO pins on the CPU.

Internal cards

Inside the enclosure there will be a base PCB that will hold 5 card slots using DDR4 DIMM connectors that users can slot expansion cards into. Our machine will have 2 base cards: a CPU card and a memory card, and a stretch goal will be a UART card. All cards will all have similar layout and use the RP2350 to emulate the functions of the card and interface with the S-100 like bus. The CPU card will connect to the front panel PCB using a cable.

Back panel

Our back panel power supply is based on X buck converter chip. it will have USB C power input and a cable to output the 3.3 V to the base board that will power all of the cards.

Project Specific Design Requirements (PSDRs):

  1. PSDR #1: (Software) Creating a Rust-based emulation of an Intel 8080 processor that is tied to the external clock/single-step on the 8080-bus.
  2. PSDR #2: (Hardware) An S-100-inspired 8080 bus protocol implemented via RP2350 PIO and hardware implementation using DDR4 DIMM connector
  3. PSDR #3: (Hardware) Creating a swtich and LED matrix for the switches on the front panel to communicate with the CPU over a connecting wire to the CPU card.
  4. PSDR #4: (Hardware) A step-down 5V -> 3.3V power supply (buck conversion) from a reference design. Input USB C output to S100 bus.
  5. PSDR #5: (Hardware) An ability to perform serial bi-directional communication via an additional UART card. We will be writing the firmware for this to run on the RP2350. It will have an external uart port on the card to plug into.
